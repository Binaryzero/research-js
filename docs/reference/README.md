# Reference Documentation

Hub for architectural decisions, development guides, operational manuals, and research.

## Directory Structure

| Directory | Purpose | Contents |
|-----------|---------|----------|
| `adrs/` | Architecture Decision Records | Significant technical decisions with context and rationale |
| `guides/` | Development guides | How-to documents for common development tasks |
| `manuals/` | Operational manuals | Runbooks, deployment procedures, troubleshooting |
| `research/` | Research notes | Technology evaluations, spike findings, benchmarks |

## ADR Format

Each ADR follows this structure:

| Section | Content |
|---------|---------|
| Title | `ADR-NNN: Decision Title` |
| Status | Proposed / Accepted / Deprecated / Superseded |
| Context | What prompted this decision |
| Decision | What was decided |
| Consequences | Trade-offs and impacts |
